# 打牌记账小程序 — 产品需求文档（PRD）

## 1. 产品定位与目标

### 1.1 产品定位
一款面向线下牌友（朋友之间）的轻量记账工具，核心解决「每局谁输谁赢、最后谁该付谁」的问题。无需注册，打开即用，数据本地存储。

### 1.2 产品目标
- **3秒开局**：选定玩家、设定初始分，快速开始
- **5秒记牌**：一指记录输赢，无需复杂操作
- **秒级结算**：局结束即出结算结果，避免口角

### 1.3 核心价值
- 消除口头算账的遗忘与争议
- 记录历史战局，回顾"谁是雀神"
- 零门槛，朋友之间扫码/分享即可使用

---

## 2. 用户画像与场景

### 2.1 典型用户
- **阿强（28岁，销售）**：周末约朋友打麻将，结束后经常为算账扯皮，想要公平公正的记录
- **王阿姨（52岁，退休）**：和邻居打牌，用纸笔记录总忘，想要简单好用的工具
- **小李（22岁，学生）**：和同学聚会玩牌，想要能显示"谁是大赢家"的统计

### 2.2 典型场景
1. 线下聚会打牌（麻将、斗地主、掼蛋等），现场记分
2. 多人轮流对战，实时排名
3. 一场多局，结束后统一结算

## 3. 用户故事

### 3.1 核心用户故事

| 编号 | 用户故事 |
|------|----------|
| US-01 | 作为牌友，我希望**快速创建房间**，邀请朋友加入，这样不用每次重复设置 |
| US-02 | 作为牌友，我希望**记录每一局的输赢**，操作要极简，这样打牌间隙不耽误时间 |
| US-03 | 作为牌友，我希望**实时看到当前排名**，这样大家随时知道局势 |
| US-04 | 作为牌友，我希望**局结束自动结算**，谁该付谁一目了然，避免尴尬 |
| US-05 | 作为牌友，我希望**查看历史战局**，和朋友比较谁赢得最多 |
| US-06 | 作为新加入的朋友，我希望**扫码就能加入**，不需要下载App或注册账号 |

### 3.2 次要用户故事

| 编号 | 用户故事 |
|------|----------|
| US-07 | 作为主持人，我希望**设置每局初始分数**，适配不同玩法 |
| US-08 | 作为主持人，我希望**设置胜负值单位**（如每分1元、10元），方便直接折算金额 |
| US-09 | 作为牌友，我希望**导出结算结果**，可以发到群里留档 |

## 4. 功能列表

### 4.1 核心功能（MVP）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建房间 | 创建游戏房间，生成房间码/二维码 | P0 |
| 加入房间 | 输入房间码或扫码加入 | P0 |
| 玩家管理 | 添加/移除玩家，设置昵称 | P0 |
| 记牌 | 记录每局每个玩家的分数变化 | P0 |
| 实时排名 | 展示当前所有玩家的排名 | P0 |
| 结算 | 局结束后计算每个人最终的应付/应收金额 | P0 |
| 本地存储 | 数据存储在本地浏览器，支持导出 | P0 |

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 历史战局 | 查看过往游戏记录 | P1 |

## 5. 非功能需求

### 5.1 性能
- 首屏加载 < 2秒
- 记牌操作响应 < 100ms

### 5.2 兼容性
- 微信小程序（首选）
- 支持 iOS 和 Android
- 兼容微信内置浏览器

### 5.3 数据安全
- 数据存储在用户本地，不上传服务器（隐私友好）
- 后续如需云同步，用户自愿开启

### 5.4 辅助功能
- 适老化：大字体、高对比度模式
- 无网络：支持离线使用

---

## 6. 产品边界

### 6.1 我们做的
- 记分、结算、排名
- 历史记录
- 简单统计

### 6.2 我们不做的
- 支付功能（不涉及真实资金流转）
- 社交功能（不建聊天/社区）
- 复杂牌局管理（不记录每局胡什么牌）
