# 打牌记账小程序 — 原型设计

## 1. 页面结构总览

```
首页
├── 快速开始（创建房间）
├── 加入房间
└── 历史战局

房间内（游戏页面）
├── 玩家列表 + 实时排名
├── 记牌区
├── 结算按钮
└── 房间信息

结算页
├── 最终排名
├── 应付/应收明细
└── 导出/分享

历史战局
├── 战局列表
└── 战局详情
```

### 2.2 房间页（Game Room）

**功能：游戏进行中的核心页面**

```
┌─────────────────────────────┐
│  房间：麻将室    房间码：384721│
│  ──────────────────────────  │
│                             │
│  👥 玩家列表（实时排名）       │
│  ┌─────────────────────────┐ │
│  │ 🥇 小王      +120       │ │
│  │ 🥈阿强       +45        │ │
│  │ 🥉小李       -35        │ │
│  │ ④ 王阿姨     -130       │ │
│  └─────────────────────────┘ │
│                             │
│  📝 记牌区                   │
│  选择本局赢家：               │
│  [小王] [阿强] [小李] [王阿姨] │
│                             │
│  本局输家：                   │
│  [小王] [阿强] [小李] [王阿姨] │
│                             │
│  输家需付给赢家：             │
│  [  1  ] [  2  ] [  5  ]   │
│                             │
│  ┌───────────────────────┐  │
│  │   ✅ 确认记牌          │  │
│  └───────────────────────┘  │
│                             │
│  ──────────────────────────  │
│  [🏠 首页]  [📊 结算]  [➕加局] │
└─────────────────────────────┘
```

**主要字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 玩家名称 | 文本 | 4-8字符 |
| 当前分数 | 整数 | 初始1000，可配置 |
| 本局得分 | 整数 | 正数=赢，负数=输 |
| 应付/应收 | 整数 | 基于规则计算 |

**交互逻辑：**

1. **选择赢家**：点击玩家头像，被选中者高亮显示（可多选，用于分摊场景）
2. **选择输家**：点击输家头像，与赢家互抵计算
3. **输入金额**：预设按钮 [1] [2] [5] [10]，也可手动输入自定义数值
4. **确认记牌**：校验赢家×输家×金额，计算每个人的得分变化，更新排名
5. **加局**：继续下一局，保留当前玩家和分数
6. **结算**：跳转结算页（需至少完成1局）

**记牌规则：**
- 赢家1人 + 输家1人：输家付给赢家 `金额`
- 赢家1人 + 输家多人：输家们平分付给赢家 `金额`
- 赢家多人 + 输家1人：输家付给赢家们平分 `金额`
- 赢家多人 + 输家多人：互抵后多退少补

**主要字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 最终分数 | 整数 | 所有局累计 |

3. 点击「导出结果」生成图片/文本，可分享到群
4. 点击「再来一局」保留玩家，重置分数，开始新战局
5. 点击「返回首页」清空当前房间，回到首页

## 3. 交互流程

### 3.1 完整游戏流程

```
[首页]
   │
   ├── [快速开始] ──→ [创建房间] ──→ [房间页]
   │                                      │
   ├── [加入房间] ──→ [输入房间码] ──→ [房间页]
   │                                      │
   └── [历史战局] ──→ [查看详情]

[房间页] ←→ 循环记牌
   │
   └── [结算] ──→ [结算页] ──→ [导出/再来一局/首页]
```

### 3.2 记牌交互流程

```
1. 选择本局赢家（点击头像高亮）
2. 选择本局输家（点击头像高亮）
3. 选择/输入金额（预设或自定义）
4. 点击「确认记牌」
   └── 系统计算分数变化
   └── 更新排名显示
   └── 清空本局选择，准备下一局
5. 可选：继续「加局」或「结算」
```

## 5. 关键交互规则

### 5.1 结算算法（最小转账路径）

为减少转账次数，采用以下算法：

1. 计算每人最终分数与平均值（初始总分/人数）的差值
2. 正差值者应收，负差值者应付
3. 按绝对值从大到小排序
4. 依次配对：最大应收 ↔ 最大应付，配平后继续

**示例（4人，初始各1000，总计4000，平均1000）：**
- 小王+320 → 应收320
- 阿强+150 → 应收150
- 小李-180 → 应付180
- 王阿姨-290 → 应付290

**配对结果：**
- 王阿姨 → 小王：290
- 小李 → 小王：30（还欠30）
- 小李 → 阿强：150

### 5.2 房间码生成规则
- 6位纯数字
- 不以0开头
- 不允许连续三位递增/递减（如123、321）
- 不允许三位重复（如111、000）
- 有效期：创建后24小时内有效

### 5.3 并发控制
- 同一房间码同一时间只允许1人操作
- 记牌后锁定300ms防止重复提交

## 6. 异常处理

| 场景 | 处理方式 |
|------|----------|
| 输入房间码不存在 | 提示"房间不存在，请检查后重试" |
| 房间已满（4人以上） | 提示"该房间人数已满" |
| 记牌时未选赢家/输家 | 按钮禁用，提示"请选择赢家/输家" |
| 输家付金额为0 | 提示"金额需大于0" |
| 断网 | 继续离线使用，重连后自动同步本地数据 |
| 页面刷新 | 本地存储恢复，不会丢失数据 |
